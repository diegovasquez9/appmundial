# PM_AI_WORKFLOW.md

## Tools & How I Used Them

**Claude (Anthropic)** — used across the full workflow. For analysis, I set up a dedicated Claude Project pre-loaded with Anthropic's native PM skill plugins (metrics review, roadmap update, stakeholder update, synthesize research) alongside the four CSV files. Having all context available simultaneously was key for cross-referencing signals across tables — matching studio verbatim feedback to CSAT scores, or trend data to RICE inputs — in a way that a one-file-at-a-time approach would have missed. Claude generated the full draft from that foundation. Sanity-checking was done by me: every number and recommendation traced back to a specific data point before it stayed in the document.

## A Workflow I'm Proud Of

Pre-loading the PM skills and all data files into a single Claude Project before writing a single line. It turned what could have been a fragmented, prompt-by-prompt exercise into one coherent analytical session grounded in the actual data.

## Where the AI Was Wrong

In the EchoVoice adoption section, Claude projected a target of "38 → 60+ active studios" — a specific number that appears nowhere in the data. Nothing in the CSVs indicates how many of the 78 non-installed studios operate in genres where VO is relevant or would realistically convert. I caught it by asking "where does that number come from?" — and it had no answer. The corrected document removes the target and flags genre segmentation as a required input before any numeric goal is set. AI fills analytical gaps with confident-sounding estimates; the sanity-check phase is where PM judgment actually happens.
